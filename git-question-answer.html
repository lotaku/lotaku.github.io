git 问题记录及解决方法

问题：在git clone或者git add remote {server} 时，报错，说证书校验有问题

#报错代码
error: server certificate verification failed. CAfile: /etc/ssl/certs/ca-certificates.crt CRLfile: none

解决方法 1 ：最简单的解决方法是调用前加一个环境变量：

$ export GIT_SSL_NO_VERIFY=1

解决方法 2 ：比1更好.

$ git config --global http.sslverify false

参考：http://blog.csdn.net/ubuntu64fan/article/details/12373403

问题：Git 每次提交时的用户名和密码设置

原因：在克隆（clone）分支时使用了 HTTPS 的地址
解决方法：切换到 SSH 方式,可以参考官方文档。

#进入本地的git项目
$ git remote set-url origin git@github.com:user/repo.git

问题:commit时出现 Permission denied (publickey).

原因： SSH key 没有设置或已经失效（譬如升级到 Mountain Lion 系统后）
解决方法：参考官方文档。
